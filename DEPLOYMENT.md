# Production Deployment — DreamScore

Step-by-step cutover from sandbox/dev to live on `unlockyourdreamscore.com` (or whichever domain).

---

## 1. Server prerequisites

- **PHP 8.2+** with extensions: `mbstring`, `openssl`, `pdo`, `pdo_mysql` (or `pdo_sqlite`), `xml`, `simplexml`, `bcmath`, `curl`, `tokenizer`, `fileinfo`
- **Composer 2.x**
- **Web server**: Apache (`mod_rewrite` enabled) or nginx with PHP-FPM
- **TLS certificate** on the production domain (Let's Encrypt is fine)
- **MySQL 8.0+** (recommended) or stay on SQLite if traffic is low

The Laravel `public/` directory must be the web-server document root — never expose the project root.

---

## 2. Deploy the code

```bash
git clone <your-repo> /var/www/dreamscore
cd /var/www/dreamscore

composer install --no-dev --optimize-autoloader

cp .env.example .env
php artisan key:generate           # writes APP_KEY
# Edit .env now — see section 3 below

php artisan storage:link
php artisan migrate --force

# Build optimized caches
php artisan config:cache
php artisan route:cache
php artisan view:cache

# Permissions (Linux)
chown -R www-data:www-data storage bootstrap/cache
chmod -R 775 storage bootstrap/cache
```

Set the web-server doc root to `/var/www/dreamscore/public`.

## 4. Webhook configuration on Authorize.Net (production)

In `https://account.authorize.net` → **Account → Settings → Webhooks**:

| Field            | Value                                                       |
|------------------|-------------------------------------------------------------|
| Endpoint URL     | `https://unlockyourdreamscore.com/authorize.net/notification` |
| Event            | `net.authorize.payment.authcapture.created`                 |
| Status           | Active                                                      |

Your screenshot shows this is already created. After cutover, fire a test event from the Authorize.Net dashboard ("Test" button next to the webhook) and confirm it appears in `storage/logs/laravel.log` as:

```
Authorize.Net webhook received   eventType=net.authorize.payment.authcapture.created
GHL payment webhook posted       status=200  sub_found=true
```

If you see `signature INVALID`, the production `AUTHORIZE_NET_SIGNATURE_KEY` doesn't match the live merchant. Regenerate it in Authorize.Net and update `.env`.

## 5. Apache `.htaccess` (already shipped in `public/`)

Laravel ships the correct `.htaccess` in `public/`. If you use Apache, ensure:

- `AllowOverride All` is set in the `<Directory>` block for `/var/www/dreamscore/public`
- `mod_rewrite` is enabled (`a2enmod rewrite`)

For nginx, the standard Laravel block is:

```nginx
server {
    listen 443 ssl http2;
    server_name unlockyourdreamscore.com;
    root /var/www/dreamscore/public;
    index index.php;

    ssl_certificate     /etc/letsencrypt/live/unlockyourdreamscore.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/unlockyourdreamscore.com/privkey.pem;

    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    # Block access to dotfiles and sensitive paths
    location ~ /\. { deny all; }
}

server {
    listen 80;
    server_name unlockyourdreamscore.com;
    return 301 https://$host$request_uri;
}
```

## 6. Trust Proxies

`app/Http/Middleware/TrustProxies.php` is set to `protected $proxies = '*'` so Laravel honors `X-Forwarded-Proto` from any upstream. **If your production server is behind a known CDN/load balancer (Cloudflare, AWS ALB, etc.), tighten this to the proxy's IP ranges.** Otherwise leave as `'*'` only if the app is *directly* terminating TLS or sitting behind a single trusted reverse proxy.

## 7. Pre-flight checklist

Before flipping DNS / launching:

- [ ] `APP_ENV=production`, `APP_DEBUG=false`, `APP_URL=https://...`
- [ ] `AUTHORIZE_NET_ENV=production` and live merchant keys in place
- [ ] Hit `/checkout/onetime` over HTTPS — Accept.js loads (no mixed-content warnings)
- [ ] Run a real $0.01 charge against a real card → succeeds → redirects to `/onboarding`
- [ ] Submit onboarding → confirm GHL receives + CRC creates a Client (check CRC dashboard → Clients tab)
- [ ] Trigger Authorize.Net webhook from dashboard test → `GHL payment webhook posted` line in log
- [ ] Refund the $0.01 in the Authorize.Net dashboard
- [ ] `php artisan config:cache && php artisan route:cache && php artisan view:cache`
- [ ] Log rotation configured (logrotate or Laravel daily channel)
- [ ] Backups configured for the database

---

## 8. Going live

1. Switch DNS / point the domain to the production server.
2. Clear any CDN caches.
3. Watch `storage/logs/laravel.log` during the first ~30 minutes:
   ```
   tail -f storage/logs/laravel.log
   ```
4. Verify the first real customer goes end-to-end: payment → onboarding → CRC client created → GHL receives both onboarding and payment-confirmation webhooks.

---

## 9. Post-launch maintenance

- **Log rotation** — Laravel's stack channel writes to `storage/logs/laravel.log`. Either switch `LOG_CHANNEL=daily` (auto-rotation) or use system `logrotate`.
- **CRC API rate limit** — your plan caps at 150 requests/day. Upgrade if you exceed.
- **Idempotency cache** — Authorize.Net webhook deduplication uses Laravel's cache for 24 h. The `file` cache driver is fine; switch to Redis if you go multi-server.
- **Clear caches after every deploy** — `php artisan optimize:clear && php artisan optimize`.
